# 요구사항 분석

	- 각 채널 혹은 영상에 대해서 필요한 데이터를 프로그래밍을 통해 수집 할 수 있어야 합니다.
	- 수집된 데이터는 전부 데이터베이스에 저장되어야 합니다.
	- 수집된 데이터 분석을 통해 의미있는 값을 도출 할 수 있어야 합니다.
	- 도출된 값을 웹 I/O를 통해 접근성을 가질 수 있어야 합니다.
	
	- 수집된 데이터를 형태소 분석기를 사용하여 명사를 추출, 사전(감정 분석을 위해서 , 인터넷에 다수 존재)을 제작한다.
	- 만들어진 사전을 바탕으로 긍정형 명사와 부정형 명사를 분석한다.
	- 댓글에서 등장한 명사들의 성격을 분석하여 댓글 자체의 성격을 분류한다.
	
	- 형태소 분석기를 돌리는 이유는 사전에 내용을 추가하기 위함이다.
	- 제품마다 사전을 만들 필요가 있다. 제품의 성격마다 특정 명사가 부정 혹은 긍정이 될 수 있기 때문이다.
	- 트래쉬 데이터는 수작업으로 어느정도 진행할 필요가 있다.
	
	- 사전이 제작되면 형태소 분석기를 다시 돌리고 명사를 추출, 이번에는 사전 제작된 사전에 기반하여 긍정,중립, 부정을 확인한다.
	- Restful API를 통한 데이터 전달 및 가공한다.
	- 형태소 분석기(자신의 자체 베이스 사전을 갖고 있음 , 명사만 분류할 수 있음)와 감성 분석을 통해서 제품의 관련 키워드 프레젠테이션.
	   
	- Nodejs , Python , AWS , React , HTML , CSS , JAVASCRIPT , LINUX

# 클래스 분석

	1. Crawling Class : 웹 페이지 데이터(html)를 긁어와서 기본적인 전처리 수행

		- 유투브 사이트의 데이터를 가져오는 메소드.
		- 가져온 데이터를 형태소 분석 클래스로 전달.( 형태소 분석 클래스 함수 콜)

	2. Morpheme analyzer Class : 형태소 분석 기능을 담당하는 클래스

		1. 형태소 분석기가 가진 기본 사전 + 특정 제품군을 위해 만들어진 사전을 이용해야 한다.
		2. 쓰레기 데이터 사전 제작 (작업할수록 알아서 걸러지는 식으로) , 자동화 지향. 

		- 기본 베이스 사전이 특정 제품군에 적용하기엔 다소 부족할 수 있으므로 커스텀할 필요가 있다.(수작업)
		- 문장에서 명사를 추출하는 메소드
		- 문장에서 트래쉬 데이터를 추출하는 메소드
		- 추출된 결과를 DB로 전달.(DB 클래스 함수 콜)

	3. DB Controller Class :  DB와 관련한 모든 트랜잭션을 담당하는 클래스
		
		1. RDBMS : CRUD 정도의 범위를 넘어서는 쿼리는 현재 요구되지 않는다.
		
		- Morpheme analyzer Class가 추출된 결과를 저장하기 위한 콜 메소드.
		- 4. Sentimental Analysis Class가 결정된 성격을 제품에 컬럼을 추가하여 기입하는 콜 메소드.
		- 원격 DB 커넥션 맺는 메소드.
		- 오류 핸들링 메소드.
		
	4. Sentimental Analysis Class : Morpheme analyzer Class가 저장한 추출된 명사를 DB Controller Class에서 쿼리를 통해 받아와서
	추출된 명사가 부정 , 긍정의 의미를 갖는지 판단하는 클래스
		
		1. 감성 분석의 기준이 되는 사전은 수작업으로 진행한다.

		- DB에서 받아온 명사를 사전 제작된 감성 분석 사전과 비교 하여 긍정 부정을 확인하는 메소드.
		- 댓글마다 (+,-)를 판단하여 댓글의 성격을 확인. 
		- 확인이 되었다면 전체 댓글의 긍정 부정 비율을 확인하여 제품 자체의 긍정 부정을 결정.
		- Aggregation과 긍정 부정 확인은 메소드 내에서 같이 결정. +@ 제품의 성격을 Column을 추가하여 기입.
		- 긍정, 부정로 나타나지 않는 단어들은 중립으로 처리한다.
